# Governance

## Current Model

This project is currently maintainer-led while the protocol is in draft stage.

## Maintainer Responsibilities

Maintainers:
- steward protocol direction,
- review and merge contributions,
- ensure compatibility and security considerations are documented,
- manage draft version progression (`v0.x`).

## Decision Process

- Minor editorial changes: maintainer review and merge.
- Normative protocol changes: issue discussion plus PR review.
- Breaking draft changes: explicit versioning notes and migration guidance.

## Versioning Direction

- `spx-prov` is currently in pre-1.0 drafts.
- Profile/binding updates may version independently (for example WebMCP profile revs).
- Major protocol-breaking changes require a major version increment.

## Future Evolution

As the project grows, governance may evolve to include a public RFC process and broader reviewer group.
