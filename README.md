# ClipRoot

**ClipRoot** is an open protocol for **provenance-aware content reuse**. It provides a structured, machine-verifiable way to capture where content comes from, how it has been transformed, and who was involved — preserving attribution as content moves between documents, tools, and agents.

The **ClipRoot Protocol (CRP)** is designed for a world where content is routinely copied, quoted, summarized, translated, and regenerated by both humans and AI. Rather than treating provenance as an afterthought, CRP makes it a first-class property of content.

## How It Works

CRP models provenance as a directed acyclic graph (DAG). The fundamental unit is a **clip** — a span of text with a content-addressed hash, source references, and optional selectors that describe where it was found. Clips link to each other through typed **edges** (inspired by the [W3C PROV Ontology](https://www.w3.org/TR/prov-o/)), forming a traceable lineage from derived content back to original sources.

```
clip (text + source) ──[summary]──► derived clip ──[paraphrase]──► another clip
  └─ sha256-abc...                    └─ sha256-def...                └─ sha256-xyz...
```

Key properties:

- **Content-addressed.** Clips and artifacts are identified by deterministic SHA-256 hashes, enabling integrity verification without a central authority.
- **Lineage-first.** Derivation relationships form a DAG, allowing any piece of content to be traced back to its original sources.
- **Agent-neutral.** Humans, AI models, and automated services are all first-class agents.
- **Local-first.** CRP bundles are self-contained JSON documents. No network or registry is required to create, verify, or exchange provenance data.
- **Schema-driven.** The canonical protocol definition is a [JSON Schema (2020-12)](schema/crp-v0.0.3.schema.json). Implementations validate against this schema, not against prose.

## Ecosystem

ClipRoot is implemented across two repositories, targeting different runtime environments:

### This Repository — Protocol, Web, and Browser Integration

This is a pnpm/Turborepo monorepo containing the canonical protocol schemas and the TypeScript/browser implementation layer.

| Package | Purpose |
|---|---|
| [`schema/`](schema/) | Canonical CRP JSON Schemas and examples |
| [`spec/`](spec/) | Human-readable protocol specification |
| [`@cliproot/protocol`](packages/protocol/) | Schema validation (AJV), TypeScript types, deterministic hashing, constants |
| [`@cliproot/core`](packages/core/) | Browser SDK for capturing clipboard provenance on copy events |
| [`@cliproot/tiptap`](packages/tiptap/) | Tiptap editor extension for span-level attribution |
| [`@cliproot/extension`](packages/extension/) | Browser extension (WXT, MV3) — captures provenance on every copy |
| [`@cliproot/playground`](apps/playground/) | Web app for inspecting clips and visualizing provenance graphs |

**Dependency chain:** `protocol` → `core` → `extension`, `tiptap` → `playground`

### [`cliproot_rust`](https://github.com/cliproot/cliproot_rust) — CLI, MCP Server, and Storage Engine

The Rust implementation provides the native runtime: a CLI with 20+ subcommands, an MCP server for AI agents, filesystem + SQLite storage, OS clipboard integration, and `.cliprootpack` archive support.

| Crate | Purpose |
|---|---|
| `cliproot-core` | Protocol model types, hashing, verification, text matching |
| `cliproot-store` | Filesystem object store + SQLite index, pack format (tar+zstd) |
| `cliproot-mcp` | MCP server (24+ tools) over stdio for AI agents |
| `cliproot-cli` | CLI binary with embedded MCP server |

Install: `cargo install --path crates/cliproot-cli` or download a pre-built binary from [Releases](https://github.com/cliproot/cliproot_rust/releases).

## Try It

### Playground

Inspect clips, explore provenance graphs, and visualize derivation lineage in the browser — no install required:

**[cliproot.github.io/cliproot](https://cliproot.github.io/cliproot/)**

### Browser Extension

Capture provenance metadata on every copy, automatically:

```bash
git clone https://github.com/cliproot/cliproot.git
cd cliproot && pnpm install
pnpm --filter @cliproot/extension dev    # Chrome
```

Load `.output/chrome-mv3/` via **chrome://extensions → Load unpacked**.

### CLI + MCP Server

Manage clips, trace lineage, and expose provenance tools to AI agents:

```bash
cargo install --path crates/cliproot-cli   # from cliproot_rust repo

cliproot init                              # initialize a .cliproot/ repository
cliproot clip --url https://example.com --quote "The key insight is provenance."
cliproot derive --from sha256-... --quote "Summary of the insight." --activity-type summary
cliproot trace sha256-...                  # show ancestor lineage
```

Register the MCP server with Claude Code:

```bash
claude mcp add cliproot -- cliproot mcp
```

See the [cliproot_rust README](https://github.com/cliproot/cliproot_rust) for the full CLI reference and MCP tool list.

## Package Details

### `@cliproot/core`

Browser-compatible API for capturing selections, building CRP clipboard bundles, and writing provenance invisibly into HTML clipboard data.

```ts
import { captureSelection, buildClipboardBundle, writeProvenanceToClipboard } from '@cliproot/core'

document.addEventListener('copy', (event) => {
  const selection = document.getSelection()
  if (!selection || !event.clipboardData) return

  const captured = captureSelection(selection, document)
  if (!captured) return

  const bundle = buildClipboardBundle({
    captured,
    documentInfo: {
      uri: window.location.href,
      title: document.title
    }
  })

  writeProvenanceToClipboard(bundle, event.clipboardData)
  event.preventDefault()
})
```

The bundle is written as a hidden `<div data-crp-bundle="...">` appended to the `text/html` clipboard entry. Plain text is left unmodified. On paste, a CRP-aware receiver can parse the attribute to reconstruct the clip's provenance.

### `@cliproot/extension`

The browser extension automates provenance capture for every page without requiring site-side integration. It supports both Chrome and Firefox.

```bash
pnpm --filter @cliproot/extension dev            # Chrome dev
pnpm --filter @cliproot/extension dev:firefox     # Firefox dev
pnpm --filter @cliproot/extension build:all       # Production builds
```

**How it works:**

1. A capture-phase copy listener snapshots the selection and builds a CRP bundle before the site's handler runs.
2. A bubble-phase listener augments whatever HTML the site wrote with the hidden provenance div, then calls `preventDefault()`.
3. If the site suppressed bubbling (`stopImmediatePropagation`), a best-effort fallback uses the Async Clipboard API (Chrome only).

**Features:** per-site controls, clip count badge, recent clips popup, options page for managing site overrides.

### `@cliproot/tiptap`

Tiptap editor extension for span-level provenance and attribution.

```ts
import { Editor } from '@tiptap/core'
import StarterKit from '@tiptap/starter-kit'
import { AttributionExtension } from '@cliproot/tiptap'

const editor = new Editor({
  extensions: [
    StarterKit,
    AttributionExtension.configure({
      onReuseDetected: (event) => {
        console.log('Reuse detected for provenance ID:', event.provenanceId)
      }
    })
  ]
})

editor.commands.setAttribution('prov_123')
```

### Playground

The playground supports three ways to load clip data:

1. **Paste** — reads `application/x-cliproot+json` from the clipboard (requires the browser extension)
2. **Open Folder** — select a directory containing `.cliproot/objects/` to load all bundles
3. **Upload JSON** — drag-and-drop or select individual CRP bundle JSON files

```bash
pnpm --filter @cliproot/playground dev      # http://localhost:5173
pnpm --filter @cliproot/playground build    # static site output in apps/playground/dist/
```
